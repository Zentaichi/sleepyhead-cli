# Versioning Policy

Sleepyhead CLI follows Semantic Versioning (SemVer).

## Patch (x.y.Z)

- Bug fixes
- Documentation improvements
- Internal refactoring
- Performance improvements

## Minor (x.Y.z)

- New CLI commands
- New supported database profiles
- New diagnostics
- New reporting capabilities
- Backward-compatible enhancements

## Major (X.y.z)

- Breaking CLI changes
- Profile schema changes
- WizardState format changes
- Backup format changes
- Report JSON schema changes
- Removal of supported platforms or database versions

Documentation-only changes do not result in a version increment.
